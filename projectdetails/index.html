Description

We will create an interactive queue that multiple users can access. In this queue, the host user will play music from the queue. Songs can be added to the queue from Spotify and possibly SoundCloud, or Youtube.

Usefulness

By allowing hosts to create events with queues that individuals can add songs to, we will be eliminating numerous pain points for groups of people who enjoy a shared-music environment. You will no longer need to search for “who is on AUX” or ask for a song to be played. You can simply add it to the event queue! The use cases for this application range from hosting social gatherings to group study sessions, road trips, and many more.

Currently, subscription-based music providers like Spotify and Apple Music allow users to create collaborative playlists, which allow multiple people to contribute to a song list at a predetermined time. However, it does not provide the functionality for an interactive queue, which allows numerous users to add songs to a playlist and decide upon the order in which they are played in real-time. Our service differs from these platforms and adds functionality that users have been requesting since 2012.

Another popular application, Rave, exists to provide similar functionality to ours, but Rave tailors to streaming videos. Partnered with YouTube and Netflix, Rave allows you to create group viewings where a host can manage suggestions from people on what to play next. Our system will be specifically for listening to music and will differ from Rave in that any user who “registers” to the event can add to the queue.

Realness

We will use a kaggle database of 232,000+ songs which includes the song title, artist, and many interesting metrics of the song. This will be the biggest source of data. Other data will be added when users host and event and people add songs to a virtual queue. Data will be able to be queried by genre and more.

Basic Features

  • Users can create events and close them when the event is over
  • When an event is created, a queue table is created to hold songs in the “queue”
  • Users can add songs to the queue
  • Songs will be removed from the queue after they are played

Advanced Features (Potential)

We have many advanced features for now and might change/expand them as we progress in the project

Feature:

Allow multiple different users to add to a single song queue

Reason:

One person can’t hog the aux

You can add songs without the social pressure of asking the music master to play the song

Feature:

Allow songs to be queued from multiple different platforms (i.e. Spotify, Youtube, Soundcloud)

Reason:

Some songs are only one one platform, especially if it is a remix

Users could already have an account on any of these platforms

Feature:

Skip and rearrange songs based on votes

Reason:

Allows for control of the mob

Feature:

Allow the host of the event to decide if explicit songs can be played

Reason:

Allows our host to have some control over the content

Feature:

Limit a song if it’s already been played at the event within the past X minutes

Reason:

Avoid repeat songs!
